Management System & Server di Kali Linux: Apache2 & Virtual Host

Linux server administration — Apache2 setup, access log analysis, and virtual host configuration

Project ini mengeksplorasi administrasi web server Linux secara end-to-end: mulai dari memverifikasi status Apache2, memonitor access log, membuat Document Root, mengonfigurasi Virtual Host untuk domain lokal, hingga memastikan website dapat diakses melalui browser.

Apache2 Kali Linux Bash Virtual Host DNS (hosts)

Project Overview

Platform
Kali Linux (romayana@roma)
Web Server
Apache2 HTTP Server
Domain Lokal
latihan.com / www.latihan.com
Document Root
/var/www/latihan.com
Browser
Firefox 140.0 (x86_64)
Status Akhir
Berhasil

1. Verifikasi Status Apache2

Langkah pertama adalah memastikan service Apache2 aktif dan berjalan sebelum melakukan konfigurasi lebih lanjut.

$ sudo systemctl status apache2
Fig 1 — Apache2 active (running) · Main PID: 4235 · Processing requests
Service apache2.service dalam kondisi active (running)
Berjalan sejak Thu 2026-05-28 23:34:07 EDT
Memory usage: 10.1M (peak: 10.4M), CPU: 199ms

2. Monitoring Access Log

2.1 Real-time Log Monitoring

Perintah tail -f digunakan untuk memantau log akses secara real-time dan melihat request HTTP yang masuk ke server.

$ tail -f /var/log/apache2/access.log
Fig 2 — Access log: GET / → 200 OK · GET /favicon.ico → 404
GET / HTTP/1.1 → Status 200, 3383 bytes — halaman utama berhasil diakses
GET /favicon.ico HTTP/1.1 → Status 404, 527 bytes — file favicon belum ada

2.2 Filter Log Error 404

$ grep "404" /var/log/apache2/access.log
Fig 3 — Error 404: /favicon.ico dan /login tidak ditemukan
GET /favicon.ico — File ikon browser tidak ditemukan (404)
GET /login — Path /login belum dikonfigurasi (404, 528 bytes)

3. Membuat Document Root

Document root adalah direktori di mana file website disimpan. Dibuat direktori khusus untuk domain latihan.com.

$ sudo mkdir /var/www/latihan.com
Fig 4 — Pembuatan direktori /var/www/latihan.com

4. Membuat File index.html

File index.html dibuat sebagai halaman utama website menggunakan editor nano. Halaman ini berisi portfolio pribadi dengan desain modern menggunakan Google Fonts dan CSS variables.

$ sudo nano /var/www/latihan.com/index.html
Fig 5 — Editor nano: index.html dengan Google Fonts dan CSS variables
Title: "Romayana - Creative Space"
Google Fonts: Syne + DM Sans
CSS Variables: color palette (--c1: #FF3CAC, --c2: #784BA0, dst.)
Layout portfolio modern dengan desain dark gradient

5. Konfigurasi Virtual Host

5.1 Menyalin Konfigurasi Default

Konfigurasi Virtual Host dibuat dengan menyalin template default Apache2, lalu memodifikasinya sesuai domain latihan.com.

$ sudo cp /etc/apache2/sites-available/000-default.conf /etc/apache2/sites-available/latihan.com.conf
Fig 6 — Menyalin konfigurasi default ke latihan.com.conf

5.2 Edit DocumentRoot

File konfigurasi dibuka dengan nano. Parameter DocumentRoot diubah dari /var/www/html menjadi /var/www/latihan.com.

$ sudo nano /etc/apache2/sites-available/latihan.com.conf
Fig 7 — VirtualHost *:80 dengan DocumentRoot /var/www/latihan.com

6. Aktivasi & Deaktivasi Site

6.1 Menonaktifkan Site Default

$ sudo a2dissite 000-default.conf
Fig 8 — Site 000-default berhasil dinonaktifkan

6.2 Mengaktifkan Site latihan.com

$ sudo a2ensite latihan.com.conf
$ sudo systemctl reload apache2
Fig 9 — Site latihan.com berhasil diaktifkan, Apache2 direload

7. Konfigurasi /etc/hosts

Karena latihan.com adalah domain lokal, ditambahkan entri pada file /etc/hosts agar sistem dapat meresolusi domain ke IP lokal (127.0.0.1).

$ cat /etc/hosts
Fig 10 — /etc/hosts: latihan.com dan www.latihan.com → 127.0.0.1
127.0.0.1
latihan.com
127.0.0.1
www.latihan.com

8. Hasil Pengujian

Setelah seluruh konfigurasi selesai, website diakses melalui browser Firefox untuk memverifikasi bahwa Virtual Host berjalan dengan benar dan halaman HTML berhasil ditampilkan.

Fig 11 — http://latihan.com berhasil diakses di Firefox 140.0
Heading: "Halo, saya Romayana" tampil dengan benar
Deskripsi, tombol "Lihat Karya" dan "Hubungi Saya" terender
Design dark gradient (pink → purple → hijau) berhasil ditampilkan
Domain menampilkan "Not Secure" karena menggunakan HTTP (bukan HTTPS)
