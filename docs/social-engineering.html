This lab demonstrates Credential Harvesting — a social engineering technique where an attacker creates a convincing clone of a legitimate website to trick users into submitting their credentials. The captured data is then logged by the attacker's server. The simulation uses SEToolkit (Social-Engineer Toolkit) running on Kali Linux.

Findings
Key Observations
Convincing Site Clone
SEToolkit's Site Cloner produced a near-identical replica of the GitHub login page, making it visually indistinguishable from the real site to an untrained user.
Real-Time Credential Capture
As soon as the victim submitted the login form, the username and password appeared instantly in the attacker's terminal, highlighted as POSSIBLE USERNAME/PASSWORD FIELD FOUND.
Persistent Log Storage
Captured credentials were saved to ~/.set/site.template, accessible later via root — demonstrating how attackers can retain stolen data even after the session ends.
No HTTPS = Clear Warning Sign
The cloned page was served over HTTP with a "Not Secure" warning in the browser — a key red flag that most users overlook when distracted or under social pressure.
Prevention
Recommendations
1
Enable Two-Factor Authentication (2FA)
Even if credentials are stolen, 2FA adds an extra verification layer (authenticator app or SMS code) that prevents unauthorized access.
2
Always Check the URL Before Logging In
Never click links blindly. Verify the full URL matches the official domain exactly. Look for HTTPS and the padlock icon — if missing, do not enter credentials.
3
Use a Password Manager
Password managers auto-fill credentials only on verified legitimate domains. If a site is a clone, the manager won't auto-fill — acting as a passive phishing detector.
4
Conduct Regular Security Awareness Training
Attackers often target people, not systems. Regular training helps users recognize phishing attempts, reducing human error and building a security-conscious culture.
